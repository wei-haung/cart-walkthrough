Why this rule?
Pre-order items ship on a future date. Mixing them with in-stock items would delay the whole order, so the cart enforces one type at a time.
Cart · Feature · HYTE Shop · Updated Apr 2026

Pre-order and standard items can't share a cart

When a shopper adds a product of a different fulfillment type to a cart that already has items, the cart pauses and asks them to confirm before replacing what's there. This walkthrough uses the actual demo to show the full flow.

Scenario
Cart holds a CNVS (pre-order). Shopper adds a Y60 (standard).
Trigger
Click Add to Cart on a different-type product.
Expected
Warning modal with Cancel and Clear Cart & Add Item.
Outcome
On replace, old item is removed and new one is added.

Three rules to remember

The behavior above is driven by three simple rules.

Rule 01
One type per cart
A cart is either all pre-order or all standard — never mixed.
Rule 02
First item sets the type
The first product added locks the cart to its fulfillment type.
Rule 03
Replacing is opt-in
A conflicting add always prompts before clearing existing items.
